Aqui está o feedback detalhado explicando o contexto de cada questão de Git:

1. **Qual comando inicializa um novo repositório Git?**
   - **A. git start**
   - **B. git create**
   - **C. git init** (RESPOSTA CORRETA)
   - **D. git new**

   **Feedback:** Para inicializar um novo repositório Git, usamos o comando `git init`. Esse comando cria um novo repositório vazio no diretório atual, configurando as estruturas necessárias para começar a usar o Git.

2. **Qual comando é usado para adicionar mudanças à área de stage no Git?**
   - **A. git add** (RESPOSTA CORRETA)
   - **B. git stage**
   - **C. git commit**
   - **D. git push**

   **Feedback:** O comando `git add` é usado para adicionar mudanças à área de stage no Git. Isso prepara os arquivos e alterações para serem incluídos no próximo commit.

3. **Como você verifica o status do seu diretório de trabalho e área de stage no Git?**
   - **A. git log**
   - **B. git show**
   - **C. git status** (RESPOSTA CORRETA)
   - **D. git check**

   **Feedback:** Para verificar o status do diretório de trabalho e da área de stage, usamos o comando `git status`. Este comando mostra quais arquivos foram modificados, adicionados à área de stage ou ainda não estão sob controle do Git.

4. **Qual comando é usado para criar uma nova branch no Git?**
   - **A. git branch new**
   - **B. git new-branch**
   - **C. git checkout -b** (RESPOSTA CORRETA)
   - **D. git create-branch**

   **Feedback:** Para criar uma nova branch e mudar para ela imediatamente, usamos `git checkout -b <nome_da_branch>`. Isso cria a nova branch e faz o checkout nela ao mesmo tempo.

5. **Qual é o comando correto para mudar para uma branch existente chamada "feature"?**
   - **A. git branch feature**
   - **B. git checkout feature** (RESPOSTA CORRETA)
   - **C. git switch feature**
   - **D. git change feature**

   **Feedback:** Para mudar para uma branch existente chamada "feature", usamos `git checkout feature`. Esse comando troca a branch atual para a branch especificada.

6. **Como você faz commit das mudanças com uma mensagem no Git?**
   - **A. git commit -m "mensagem"** (RESPOSTA CORRETA)
   - **B. git commit "mensagem"**
   - **C. git add -m "mensagem"**
   - **D. git push -m "mensagem"**

   **Feedback:** Para fazer commit das mudanças com uma mensagem, usamos `git commit -m "mensagem"`. A opção `-m` permite incluir uma mensagem descritiva do commit diretamente no comando.

7. **Qual comando você usa para mesclar uma branch chamada "feature" na branch atual?**
   - **A. git combine feature**
   - **B. git merge feature** (RESPOSTA CORRETA)
   - **C. git join feature**
   - **D. git integrate feature**

   **Feedback:** Para mesclar uma branch chamada "feature" na branch atual, usamos `git merge feature`. Este comando une as mudanças da branch "feature" na branch em que você está atualmente.

8. **Como você visualiza o histórico de commits no Git?**
   - **A. git log** (RESPOSTA CORRETA)
   - **B. git history**
   - **C. git show**
   - **D. git commits**

   **Feedback:** Para visualizar o histórico de commits, usamos `git log`. Este comando exibe uma lista dos commits feitos no repositório, com detalhes como autor, data e mensagem de commit.

9. **Qual comando você usa para clonar um repositório do GitHub?**
   - **A. git copy**
   - **B. git fork**
   - **C. git clone** (RESPOSTA CORRETA)
   - **D. git download**

   **Feedback:** Para clonar um repositório do GitHub, usamos `git clone <URL_do_repositório>`. Isso cria uma cópia local do repositório remoto.

10. **Como você envia seus commits para um repositório remoto no Git?**
    - **A. git pull**
    - **B. git push** (RESPOSTA CORRETA)
    - **C. git upload**
    - **D. git send**

    **Feedback:** Para enviar seus commits para um repositório remoto, usamos `git push`. Este comando envia as mudanças locais para a branch correspondente no repositório remoto.
